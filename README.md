## Запуск проекта

```bash
python -m src.db
```

## Запуск тестов

```bash
python -m unittest discover     
```

## Что умеет проект

- добавление мемов в память или JSON-файл
- выбор способа хранения при запуске программы
- поиск мемов по одному или нескольким полям
- обновление существующих записей
- удаление мемов по ID
- просмотр всех записей
- сохранение происходит после каждой операции изменения

## Структура проекта

```text
src/db/backend/record.py  - класс записи
src/db/backend/table.py   - работа со списком записей
src/db/backend/database.py - общий интерфейс базы данных
src/db/backend/memory.py  - класс базы данных в памяти
src/db/backend/file.py    - класс файловой базы данных
src/db/tui.py             - текстовое меню с выбором способа хранения
tests/test_memory.py      - тесты базы в памяти
tests/test_file_database.py - тесты файловой базы
```

## Коротко о реализации

Каждый мем хранится как объект `Record`, внутри таблицы записи лежат в обычном списке.
Интерфейс `Database` задает основные методы базы данных: добавление, поиск, обновление и удаление.
Класс `Memory` хранит данные только во время работы программы.
Класс `FileDatabase` наследуется от того же интерфейса, но сохраняет таблицу и записи в файл `data/memes.json` в формате JSON.
